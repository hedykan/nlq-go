# LLM 性能优化测试报告

> 测试时间: 2026-03-28  
> 测试模型: GLM-4.7  
> 测试工具: langchaingo v0.1.14

---

| 配置 | 参数 | 中文查询 | 英文查询 | 提升 |

## 详细测试结果

### 1. 基准测试 (Baseline)

```
配置: Temperature=0.1, MaxTokens=1000
中文查询: 91秒
英文查询: 26秒
```

### 2. Temperature 调优

| Temperature | 响应时间 | 提升倍数 |
|-------------|---------|---------|
| 0.0 (贪心) | 12.8秒 | **7x** |
| 0.1 (当前) | 91秒 | 基准 |
| 0.2 | 10.5秒 | **8.7x** |

**结论**: Temperature=0.0 或 0.2 都能显著提升速度，Temperature=0.2 略快。

### 3. MaxTokens 调优

| MaxTokens | 响应时间 | 提升倍数 |
|-----------|---------|---------|
| 1000 | 12.8秒 | 基准 (temp=0.0) |
| 2048 | 9秒 | **1.4x** |
| 4096 | 52.9秒 | **0.24x** (变慢!) |

**重要发现**: MaxTokens=4096 反而更慢，可能原因：
- 模型生成更多内容需要更多时间
- 或触发了一些内部优化机制

**结论**: MaxTokens=2048 是最佳选择。

### 4. Thinking 模式测试

| Thinking | 响应时间 | 提升倍数 |
|----------|---------|---------|
| 启用 (默认) | 9秒 | 基准 |
| 禁用 | 11.9秒 | **0.76x** (变慢!) |

**重要发现**: GLM-4.7 的思考模式已经高度优化，禁用后反而更慢！

**结论**: 保持 Thinking 启用状态。

### 5. TopP 调优

| TopP | Temperature | 响应时间 |
|------|-------------|---------|
| 0.0 (禁用) | 0.1 | 91秒 (基准) |
| 0.9 | 0.1 | 11.9秒 |

**结论**: TopP=0.9 配合 Temperature=0.1 有明显提升。

## 推荐配置

### 最优性能配置

```go
Temperature := 0.0   // 贪心策略，最快
MaxTokens := 2048   // 足够生成SQL，避免过大
Thinking := 启用     // GLM-4.7 已优化，禁用反而慢
```

**效果**: 中文查询从 **91秒 → 9秒** (10倍提升)

### 最优准确性配置

```go
Temperature := 0.1   // 轻微随机性
MaxTokens := 2048   // 足够生成复杂SQL
TopP := 0.9         // 配合temperature使用
Thinking := 启用     // 更准确的推理
```

---

## 关键发现

### 1. Temperature 影响巨大
- 0.0 (贪心) 比 0.1 (当前) 快 **7-10倍**
- 对于SQL生成任务，确定性输出更合适

### 2. MaxTokens 存在拐点
- 2048 是最佳值
- 过大(如4096)反而更慢

### 3. Thinking 模式已优化
- GLM-4.7 的思考模式经过优化
- 禁用不会提升速度，反而降低

### 4. TopP 与 Temperature 互斥
- 同时设置只会有一个生效
- 建议只使用 Temperature
